ACG Wireframes
Click on thumbnails to view larger wireframe
Wireframe notes: These wireframes explore information architecture and indicate a general approach towards establishing layout and user interface conventions. Most text and graphical elements are abstracted in favor of broad conceptual blocks and constructors. This isn't final design but rather aims to inform the design process.
Home Page

  • Ancillary Menu

    This is a small header element that contains utility links (About, Join, etc.) and also has a user widget. This is a global element across all ACG web properties.

  • Primary Menu

    In addition to establishing the main top-level menu items, the primary menu contains a Menu Icon (aka "Hamburger Menu") that toggles a Mega Menu State and a search icon that toggles a Search State.

  • Home Hero Area

    Proposed: an interactive area that allows the user to identify with one of three User Profiles and obtain high level content and Call-to-Actions suited for that user (content TBD).

  • Home Secondary Area

    High-level touts to commonly trafficked portions of the ACG web ecosystem. These can be updated and shifted according to content strategy.

  • Home ACG Blog Area

    Three tiles that represent either latest Posts to the ACG Blog or curated Posts depending on editorial strategy.

  • Sites Carousel

    Allows users to cycle through premier ACG web properties to gain an understanding and also obtain an update for each. Content TBD and will most likely vary depending on the scope of content and the intent.

  • Home Tertiary Area

    Introduces a tile/grid of components, widgets, and editorial content that allows the user to scan and deep-dive into various properties of the ACG web ecosystem.

Mega Menu Toggle/State

  • Full Screen Overlay on Toggle

    The Menu/Hamburger Icon toggles onClick (or onTouch in a mobile context) a deep-dive menu. My feeling is it would be reinstatement of the primary navigation, with the entire Secondary submenus for Primary items (or a curated list). The mobile version would scroll on touch, and collapse into two columns or a single column.

    The bottom left portion of the Menu Toggle could be reserved to enumerate the items not represented in the primary navigation (or perhaps buried for organizational purpose but in actuality could be given more visibility). The bottom right portion reserved for high level call-to-action, and can be considered optional for mobile due to space/scroll restraints).

Section Page

  • Hero Area

    Simple headline and introductory statement overlayed over background image/stock photo that matches that Section (in this case could be a Trainee-related image).

  • Section Tiles

    Re-states the grid/tiles pattern established on the Home Page, that allows the user to scan and deep-dive into various sub-Sections included within this section.

Individual Page

Common elements introduced

  • Sub-sectional menu to allow users to jump quickly to sub-Sections within a particular Section.

  • Breadcrumb to introduce a scent of information, and allow the user to traverse up the site tree.

  • 3rd-level menu system (if needed) centered horizontally above content that gives the user an idea of their current place, and allows them to toggle other sub-sub-Sections of the current sub-Section.

Fixed Header/Primary Navigation OnScroll

Re-introduce the Header/Primary Navigation

When the user scrolls past a certain preliminary point in the page, we re-introduce the Header/Primary Navigation by sliding this element into a fixed position in the viewport. The branding/logo is shrunk down so the height of this element doesn't obscure too much of the page. The background can be either reversed or lightened to avoid being too intrusive.

Search Toggle State

Search is hidden by default, but easily accessed with click/touch

The user activates a visually larger and expanded (in scope) search widget. The scope of the search can be explicitly controlled by the user to either search all of the Gi.org website or specific sub-sites or another isolated scope (perhaps for instance the ACG Blog)?
